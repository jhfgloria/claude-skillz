### Model
- Use Haiku 4.5 while commiting code.

### Base Rules
- Keep the subject line succinct.
- No need for further description in the body.
- Use 3rd person Simple Present to describe the work (e.g. "Uses ...", "Creates ...", "Adds ...").
- Avoid more than 50 characters, unless really needed to describe the work.
- If it is the first commit of the branch use the prefixes (fix:, chore:, feat:, sec:) before the description.
- No sign-offs — don't add Co-Authored-By or other signing lines.
